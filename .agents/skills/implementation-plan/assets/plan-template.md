## TDD 方針

- 最初に作るのはテストコード一式ではなく、テストリスト / 不安リストである
- 各Stepでは、テストリストから次に扱うテストを1つだけ選ぶ
- Red / Green / Refactor を混ぜない
- Greenでは、選んだ1テストを通す最小実装に集中する
- Refactorでは、全テストが通る状態を保ったまま、実装コードとテストコードを整理する
- 実装中に見つかった追加観点は、その場で実装せずテストリストへ戻す
- テストは実装構造ではなく、外から見た振る舞いに寄せる
