# Test Coverage Guide

Plan の各 Step で TDD を実践する際、テストケースは以下の 6 カテゴリをすべてカバーすること。

## カテゴリ一覧

| カテゴリ | 説明 | 例 |
|---|---|---|
| **正常系** | 主なユースケース | `handle_status_shows_info`, `handle_model_switch` |
| **空・ゼロ状態** | 入力なし・データなし・初期状態 | `handle_compact_empty_session`, `list_skills_empty` |
| **境界値** | ギリギリの値・制限・閾値 | `force_compact_runs_regardless_of_threshold` |
| **異常系** | エラー・失敗パス・存在しない対象 | `clear_session_idempotent_on_empty_chat` |
| **エッジケース** | 想定外の入力・形式 | 前後空白、大文字小文字、空文字、特殊文字、null、未定義コマンド |
| **統合** | モジュール間の繋ぎ込み・チャネル別動作 | `telegram_slash_does_not_enter_agent_loop` |

## カテゴリ別のテスト設計指針

### 正常系
- 各コマンド・各関数の基本パス
- 「正しい入力 → 期待する出力」の検証
- 副作用の確認（DB書き込み、ファイル生成等）

### 空・ゼロ状態
- 空の入力、空のコレクション、データなしの状態
- 新規作成直後の初期状態
- 「データがない」ことがエラーにならないこと

### 境界値
- 閾値の ±1、最大値、最小値
- 「閾値をバイパスする」処理の確認（`force_compact` 等）
- 制限に引っかかる・引っかからないの境界

### 異常系
- 存在しない ID、不正な引数
- DB エラー、ネットワークエラーのハンドリング
- 冪等性の確認（同じ操作を2回実行してもエラーにならない）

### エッジケース
- 入力形式のバリエーション: 大文字小文字、空白（前後・連続・タブ）、null、空文字
- 文字エンコーディング: 絵文字、マルチバイト文字、特殊文字
- メンション形式: `@bot /cmd`、`@bot/cmd`、`<@U123> /cmd`、複数空白
- コマンドの変形: `/STATUS`、`/status extra args`、`//`（コードコメントとの区別）

### 統合
- チャネル別の動作: テキストインターセプト、Interaction ハンドラ
- モジュール間の繋ぎ込み: コマンド → DB → レスポンス の一連の流れ
- 「エージェントループに入らない」ことの確認（スラッシュコマンドの核心）
